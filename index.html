Custom Programming Language Documentation

                
                

                    
Language Overview

                    
This custom programming language is designed to demonstrate core compiler concepts including tokenization, parsing, semantic analysis, intermediate code generation, and optimization. It features a syntax similar to C-style languages, supporting basic constructs such as variable declarations, arithmetic operations, control flow, and input/output.
                        


                        The language is compiled into Python through a multi-phase process that includes optimization using Reinforcement Learning (RL). This allows the compiler to intelligently apply transformations such as constant folding, dead code elimination, and common subexpression elimination, resulting in cleaner and more efficient target code.
                        


                        It is built primarily for educational and experimental purposes, providing real-time visualization of each compilation stage and allowing users to understand how high-level code evolves into optimized executable logic.

Compiler Features

                    
    
    
  • Lexical Analysis: Scans the input source code and converts it into a stream of tokens, identifying keywords, identifiers, operators, and symbols.
    • 
    
  • Parsing: Constructs an Abstract Syntax Tree (AST) from the token stream to represent the syntactic structure of the program.
    • 
    
  • Semantic Analysis: Validates the AST by checking for semantic errors like undeclared variables, type mismatches, and improper control flow.
    • 
    
  • Intermediate Code Generation: Translates the AST into a lower-level representation(three-address code), suitable for analysis and optimization.
    • 
    
  • RL Optimization: Applies reinforcement learning techniques to improve code by performing transformations like constant folding, dead code elimination, and subexpression reuse.
    • 
    
  • Code Generation: Converts the optimized intermediate code into clean and executable Python code that preserves the original program logic.
    • 
    
  • Secure Execution: Safely runs the generated Python code in a sandboxed environment and displays real-time output or error messages to the user.
    • 



                    
Optimization Features

                    
The compiler uses reinforcement learning to intelligently optimize the intermediate code by applying the following techniques:


    
    
  • Constant Folding: Detects and evaluates constant expressions during compilation to reduce runtime computation.
    • 
    
  • Dead Code Elimination: Identifies and removes code that has no impact on the final output, such as assignments to unused variables or redundant operations.
    • 
    
  • Common Subexpression Elimination: Detects duplicate expressions that compute the same value and replaces them with a single variable reference to avoid repeated calculations.
